># 专注时长记录接口文档

---

>## 开始专注
开始专注时，客户端向服务端发送一个请求，后端发送一个会话并记录下时间

>### 请求方式
POST
>### 请求参数
无参数
>### 返回实例
    {
		"code":    "200",
		"msg":     "开始记录专注数据",
		"session": guid,
	}
>## 心跳数据
为了确保记录的是专注状态，客户端每间隔一分钟向服务端发送一个请求，后端会延长会话的时间
如果在2分钟之内没有连接，后端将会记录专注时间为最后连接时的时间
>### 请求方式
PUT
>### 请求参数
|参数名|参数类型|是否可空|参数说明|
|---|---|---|---|
|session|string|否|会话的id|
>### 返回参数
|参数名|参数类型|是否可空|参数说明|
|---|---|---|---|
|code|int|否|状态码|
|msg|string|否|状态信息|
>### 返回示例

    {
		"code": "200",
		"msg":  "打卡成功",
	}

>### 状态码
|状态码|信息|
|---|---|
|200|数据交互成功|
|300|会话无效|
|500|未知错误|
>## 结束专注 
结束专注时，客户端向服务端发送一个请求，后端会关闭当前会话，并记录时间
>### 请求方式
DELETE
>### 请求参数
|参数名|参数类型|是否可空|参数说明|
|---|---|---|---|
|session|string|否|会话的id|
>### 返回参数
|参数名|参数类型|是否可空|参数说明|
|---|---|---|---|
|code|int|否|状态码|
|msg|string|否|状态信息|
>### 返回示例

>### 状态码
|状态码|信息|
|---|---|
|200|数据交互成功|
|300|会话无效|
|500|未知错误|
